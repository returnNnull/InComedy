# Статус реализации

Этот документ хранит текущий delivery status и ближайшие bounded product contexts.

Используй его, когда нужно:

- быстро понять, что уже доставлено в репозитории;
- увидеть, что ещё частично реализовано;
- не смешивать стабильную архитектуру и оперативный delivery status в одном документе.

## Части (точный порядок)

1. `implementation-status/implementation-status-part-01.md` (current delivery status and near-term bounded contexts)

## Правило добавления

- Append current delivery status updates to `implementation-status/implementation-status-part-01.md`.
- If the latest part grows above ~8,000 characters, create `implementation-status/implementation-status-part-02.md`, update this index, and continue appending there.

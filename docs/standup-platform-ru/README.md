# InComedy: продуктовая и техническая спецификация

Статус: рабочая спецификация для проектирования и реализации.

Этот пакет документов описывает целевой продукт InComedy как платформу для организации стендап-мероприятий с тремя основными ролями: комик, организатор, зритель.

Документы написаны на русском и предназначены для:
- product owner / founder;
- backend-разработчиков;
- Android-разработчиков;
- iOS-разработчиков;
- QA / аналитиков;
- дизайнеров и подрядчиков по интеграциям.

## Базовые продуктовые решения

- Один аккаунт может иметь несколько ролей. Роли не должны быть взаимоисключающими.
- Основная ценность MVP находится в организаторских операциях, рассадке, продажах билетов, лайнапе и live-управлении шоу.
- Полноценный публичный чат аудитории не должен быть обязательной частью MVP. Для запуска важнее event feed, уведомления и live-статусы.
- Авторизация через Telegram, VK и Google допустима, но для iOS-релиза нужно добавить `Sign in with Apple`.
- Билеты на офлайн-мероприятия должны продаваться через внешний PSP/merchant flow, а не через store billing.
- Донаты комикам требуют отдельной юридической и платформенной модели; их нельзя проектировать как “ещё один обычный платеж”.
